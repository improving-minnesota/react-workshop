Lecture 01

NPM

npmjs.org

NPM is the official package manager for Node.js

  • NPM runs on the command line and manages dependencies for an application
  • It also allows users to install Node.js applications that are available on the NPM registry

{
  "name": "my-project",
  "version": "0.0.1",
  "description": "My Cool Node Project",
  "main": "index.js",
  "author": "Sterling Archer",
  "dependencies": {
    "grunt": "0.4.2"
  }
}
  • When checking out a fresh project from GitHub, simply run: 'npm install' to install all the dependencies required to run the project
  • You can visit the NPM website to learn more about them

Browserify

browserify.org

  • Browserify lets you require('modules') in the browser by bundling up all of your dependencies
  • We can use Node.js modules in the browser
  • No need for two different module systems anymore

var React = require('react/addons');
var myModule = require('./modules/myModule');
  • Browserify will read .js extensions by default
  • You can also configure it to read .jsx extensions

% browserify main.js

Reactify

github repo

  • Browserify transform for JSX (superset of JavaScript used in React library)

% browserify -t reactify main.js

Gulp

gulpjs.com

  • The streaming build system
  • Runs tasks asynchronuously and uses streams

gulp.task('uglify', ['watchify'], function () {
  return gulp.src(dist('/js/app.js'))
    .pipe(gulp.dest(dist('/js')))
    .pipe(uglify())
    .pipe(rename({extname: '.min.js'}))
    .pipe(gulp.dest(dist('/js')))
    .pipe(livereload())
    .on('error', gutil.log.bind(gutil, 'Error during minification.'));
});

Mocha

mochajs.org

  • Feature-rich JavaScript test framework running on node.js and in the browser

BDD-style (Behavior Driven Development)


describe('when there is no message', function () {
  beforeEach(function () {
    Snackbar = require('./snackbar');
    element = TestUtils.renderIntoDocument(<Snackbar />);
  });

  it('should hide the snackbar', function () {
    var div = TestUtils.findRenderedDOMComponentWithTag(element, 'div');
    expect(div.props.className).contains('hide');
  });
});

Proxyquire

Proxyquire Github repo

  • Proxies Node's require so we can provide mock dependencies during testing while staying totally unobtrusive

proxies = {
  './employee.form': mockComponent('EmployeeForm'),
  '../../actions/employee.actions': {
    get: sinon.stub(),
    update: sinon.stub()
  },
  'react-router': {
    RouteHandler: mockComponent('RouteHandler'),
    Link: mockComponent('Link'),
    State: {
      getParams: function () {return {_id: 'abc123'}}
    }
  }
};

EmployeeDetail = proxyquire('./employee.detail', proxies);
